#
আপনার প্রশ্ন দুটি খুবই যৌক্তিক এবং গুরুত্বপূর্ণ। আমি দুটি বিষয়েরই বিস্তারিত ব্যাখ্যা এবং ধাপে ধাপে নির্দেশিকা নিচে দিচ্ছি।

### **প্রশ্ন ১: `crontab -e` ফাইলে নতুন কমান্ড যোগ করা**

**উত্তর: হ্যাঁ, ১০০% হবে।**

`crontab -e` কমান্ডটি আপনার বর্তমান ব্যবহারকারীর (এই ক্ষেত্রে `root`) জন্য নির্ধারিত কাজ বা "cron jobs"-এর একটি তালিকা খোলে। এই ফাইলটিতে আপনি যতগুলো খুশি আলাদা আলাদা কাজ যোগ করতে পারেন। প্রতিটি কাজ একটি নতুন লাইনে লিখতে হয়।

আগের কাজের নিচে নতুন লাইন যোগ করাটাই হলো সঠিক এবং আদর্শ পদ্ধতি।

**উদাহরণস্বরূপ:**

ধরুন, `crontab -e` খোলার পর আপনার ফাইলটি দেখতে এমন:

```
# This is a comment, it does nothing
# m h  dom mon dow   command

# পুরানো কোনো কাজের কমান্ড
15 4 * * * /usr/bin/some_other_script.sh
```

আপনি কোনো কিছু পরিবর্তন না করে, ফাইলের একেবারে শেষে আমাদের নতুন অটো ব্যাকআপের লাইনটি যোগ করে দেবেন। ফাইলটি তখন দেখতে এমন হবে:

```
# This is a comment, it does nothing
# m h  dom mon dow   command

# পুরানো কোনো কাজের কমান্ড
15 4 * * * /usr/bin/some_other_script.sh

# আমাদের নতুন ফাইল ব্রো বটের অটো ব্যাকআপ কমান্ড
0 3 * * * /root/FileBroBot/venv/bin/python3 /root/FileBroBot/auto_backup.py >> /root/FileBroBot/cron.log 2>&1
```

এটি করার ফলে আপনার আগের কাজটিও চলবে এবং আমাদের নতুন ব্যাকআপের কাজটিও প্রতিদিন ভোর ৩টায় চলবে। একটির সাথে আরেকটির কোনো সংঘর্ষ হবে না।

### **প্রশ্ন ২: ডেটাবেজ রিস্টোর কীভাবে করবেন?**

ডেটাবেজ রিস্টোর করার প্রক্রিয়াটি সরাসরি টেলিগ্রাম থেকে না করে সার্ভার থেকে করাটাই সবচেয়ে নিরাপদ। এর জন্য আমরা যে `restore.py` স্ক্রিপ্টটি তৈরি করেছি, সেটি ব্যবহার করতে হবে।

এখানে সম্পূর্ণ প্রক্রিয়াটি ধাপে ধাপে দেওয়া হলো:

#### **ধাপ ১: ব্যাকআপ ফাইলটি হাতে নিন**

*   আপনার বটের অটোমেটিক ব্যাকআপগুলো আপনার নির্ধারিত **ব্যাকআপ চ্যানেলে** (`BACKUP_CHANNEL_ID`) জমা হবে।
*   প্রথমে সেই প্রাইভেট চ্যানেল থেকে যে দিনের ব্যাকআপটি রিস্টোর করতে চান, সেই `backup_...json` ফাইলটি আপনার **লোকাল কম্পিউটারে (ডেস্কটপ/ল্যাপটপ)** ডাউনলোড করুন।

#### **ধাপ ২: বট সার্ভিস বন্ধ করুন (সবচেয়ে গুরুত্বপূর্ণ)**

রিস্টোর করার সময় ডেটাবেজে যাতে নতুন কোনো লেখা না হয়, সেজন্য বটটিকে অবশ্যই বন্ধ করতে হবে।
```bash
sudo systemctl stop filebrobot
```

#### **ধাপ ৩: ব্যাকআপ ফাইলটি সার্ভারে আপলোড করুন**

*   আপনার লোকাল কম্পিউটার থেকে ডাউনলোড করা `backup_...json` ফাইলটি এখন সার্ভারের `FileBroBot` ফোল্ডারের ভেতরে আপলোড করতে হবে।
*   এর জন্য আপনি **FileZilla**-এর মতো কোনো সফটওয়্যার ব্যবহার করতে পারেন অথবা `scp` কমান্ড ব্যবহার করতে পারেন।

#### **ধাপ ৪: রিস্টোর স্ক্রিপ্টটি চালান**

এখন সার্ভারের টার্মিনালে প্রবেশ করে নিচের কমান্ডগুলো একটির পর একটি চালান।

1.  **প্রোজেক্ট ফোল্ডারে যান:**
    ```bash
    cd /root/FileBroBot
    ```

2.  **ভার্চুয়াল এনভায়রনমেন্ট অ্যাক্টিভেট করুন:**
    ```bash
    source venv/bin/activate
    ```
    (টার্মিনালের শুরুতে `(venv)` লেখা দেখতে পাবেন।)

3.  **রিস্টোর স্ক্রিপ্টটি চালান:**
    ```bash
    python3 restore.py
    ```

#### **ধাপ ৫: স্ক্রিপ্টের নির্দেশ অনুসরণ করুন**

*   স্ক্রিপ্টটি প্রথমে আপনার কাছে ব্যাকআপ ফাইলের পাথ বা নাম চাইবে। আপনি যে ফাইলটি আপলোড করেছেন, তার সঠিক নাম দিন (e.g., `auto_backup_2025-08-08.json`) এবং **Enter** চাপুন।

*   এরপর স্ক্রিপ্টটি ফাইলটি পরীক্ষা করে দেখবে যে এটি একটি বৈধ ব্যাকআপ ফাইল কিনা (স্বাক্ষর মিলিয়ে দেখবে)।

*   ভেরিফিকেশন সফল হলে, আপনাকে একটি **চূড়ান্ত সতর্কবার্তা** দেখানো হবে। রিস্টোর নিশ্চিত করার জন্য আপনাকে `YES I AM SURE` হুবহু টাইপ করে **Enter** চাপতে হবে।

*   আপনি নিশ্চিত করার সাথে সাথেই স্ক্রিপ্টটি বর্তমান ডেটাবেজ মুছে ফেলে ব্যাকআপ থেকে নতুন ডেটা দিয়ে দেবে এবং শেষে আপনাকে একটি সফল হওয়ার বার্তা দেখাবে।

#### **ধাপ ৬: বট সার্ভিস আবার চালু করুন**

রিস্টোর প্রক্রিয়া সফলভাবে সম্পন্ন হওয়ার পর, বটটিকে আবার আগের মতো চালু করে দিন।
```bash
sudo systemctl start filebrobot
```

এখন আপনার বটটি রিস্টোর করা ডেটাবেজ নিয়ে চলতে শুরু করবে। এই নিরাপদ পদ্ধতিটি আপনাকে ভুলবশত ডেটা হারানোর ঝুঁকি থেকে রক্ষা করবে।
